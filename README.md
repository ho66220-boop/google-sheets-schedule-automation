# Google Sheets 기반 학원 출결 스케줄 자동화

## Overview

GPT와 Google Sheets, Apps Script를 활용해 학생 수강 시간표 생성과 출석 시트 반영을 자동화한 프로젝트입니다.

기존에는 학원 관리 시스템에서 학생별 수강 정보를 직접 검색하거나 전체 엑셀을 내려받은 뒤, 직원이 수강기호를 해석해 수작업으로 시간표를 정리해야 했습니다. 이 프로젝트는 관리 시스템에서 내려받은 엑셀 데이터를 기반으로 수강기호, 교실, 좌석, 수강 시작일, 휴강일, 종강일을 자동 해석하여 교실별 시간표와 출석 시트에 반영합니다.

## Background

학생 단과 수강 여부를 확인하려면 학원 관리 시스템에서 학생 이름을 일일이 검색하거나, 전체 수강생 엑셀 파일을 다운로드해야 했습니다.

전체 엑셀 파일에는 모든 학생이 포함되어 있어 필요한 학생만 분류해야 했고, 수강기호에 따라 요일과 교시를 다시 해석해야 했습니다. 엑셀 활용에 익숙하지 않은 직원들은 결국 학생별 스케줄을 직접 확인하고 입력했으며, 주마다 약 30명 정도의 학생 스케줄을 확인하는 데 30분가량이 소요되었습니다.

또한 학생에게는 단과뿐 아니라 타학원, 정기 외출, 지각, 조퇴 등 다양한 일정이 존재했습니다. 이런 일정은 각 반 담당자가 따로 관리하다 보니 공용 출결 시트에 늦게 반영되거나 누락되는 경우가 있었습니다. 그 결과 학생이 부재했을 때 부재 사유를 즉시 파악하기 어려웠고, 학부모 응대나 부재 문자 발송도 늦어질 수 있었습니다.

## Problem

- 학생별 단과 수강 정보를 수작업으로 검색하고 정리해야 했습니다.
- 관리 시스템 엑셀 데이터의 수강기호를 사람이 직접 해석해야 했습니다.
- 직원별 엑셀 숙련도에 따라 업무 속도와 정확도가 달라졌습니다.
- 타학원, 정기 일정 등 별도 스케줄이 공용 출결 시트에 늦게 반영될 수 있었습니다.
- 단과와 타학원 일정이 충돌해도 즉시 확인하기 어려웠습니다.

## Goal

반복적인 검색과 수기 입력을 줄이고, 관리 시스템에서 내려받은 엑셀 데이터만 넣으면 각 반 구조에 맞게 학생 시간표가 자동 생성되도록 만드는 것이 목표였습니다.

추가로 생성된 시간표가 실제 출결 시트에 바로 반영되도록 하고, 타학원과 같은 정기 일정도 함께 관리하여 학생 관리 정보를 한곳에서 공유할 수 있도록 설계했습니다.

## Sheet Structure

| Sheet | Role |
|---|---|
| 기본 | 학생의 학번, 학년, 호실, 좌석 등 기본 정보 관리 |
| 단과수강생 | 관리 시스템에서 다운로드한 엑셀 데이터를 붙여넣는 시트 |
| 타학원생 관리 | 타학원, 정기 외출 등 단과 외 고정 일정 관리 |
| 교실별 시트 | 학생별 실제 주간 시간표 확인 |
| 충돌확인 | 단과와 정기 일정이 겹치는 경우 기록 |
| 시간표(본관), 시간표(2관) | 학생에게 보여주기 위한 출력용 시간표 |
| 출석 시트 | 실제 출결이 이루어지는 외부 시트 |

## Key Features

### 1. 수강기호 기반 시간표 자동 생성

관리 시스템 엑셀의 수강기호를 파싱하여 요일과 교시를 해석하고, 학생의 교실 및 좌석 정보에 맞게 각 교실별 시간표를 자동 생성했습니다.

### 2. 타학원 및 정기 일정 반영

단과 외에도 타학원, 정기 외출, 지각 등 반복 일정이 존재하기 때문에 별도 관리 시트를 만들고, 해당 일정도 시간표에 함께 반영되도록 구성했습니다.

### 3. 충돌 자동 감지

단과 수업과 타학원 일정이 같은 요일/교시에 겹칠 경우 `충돌확인` 시트에 자동 기록되도록 했습니다. 이를 통해 학생에게 잘못된 스케줄을 빠르게 안내할 수 있습니다.

### 4. 출석 시트 자동 반영

생성된 시간표를 실제 출석 시트에 반영하도록 Apps Script를 작성했습니다. 출석 시트에는 오늘부터 7일간의 일정만 반영되도록 구성했습니다.

### 5. 날짜 예외 처리

수강 시작일, 종강일, 휴강일이 학생마다 다르기 때문에 출석 반영 단계에서 날짜 조건을 적용했습니다. 시간표 생성은 전체 주간 구조를 유지하고, 실제 출석 반영에서만 개강 전/종강 후/휴강일을 제외하도록 분리했습니다.

### 6. 보호값 처리

출석 시트에는 직원이 직접 입력한 지각, 병결, 조퇴, 외출 등의 값이 존재합니다. 자동화가 이런 수기 입력값을 덮어쓰지 않도록 보호값 목록을 두고, 자동 입력값만 갱신하거나 삭제하도록 설계했습니다.

## Technical Challenges

가장 어려웠던 부분은 예외 처리였습니다.

수강 시작일, 휴강일, 종강일이 학생마다 달랐고, 직원들이 타학원 일정이나 정기 일정을 입력하는 방식도 일정하지 않았습니다. 예를 들어 같은 의미의 일정도 요일과 교시를 적는 방식이 다를 수 있었기 때문에, 다양한 입력 형식을 받아들일 수 있는 파싱 규칙을 만들어야 했습니다.

또한 시간표 생성과 출석 반영의 기준을 분리하는 것도 중요한 설계 포인트였습니다. 전체 주간 시간표는 학생의 기본 수강 구조를 보여줘야 했고, 실제 출석 시트에는 날짜 조건에 따라 당일 기준으로 유효한 수업만 반영되어야 했습니다. 그래서 시간표 자동화에서는 날짜 조건을 적용하지 않고, 출석 반영 로직에서만 개강일/종강일/휴강일을 적용했습니다.

## Result

- 주당 약 30분 걸리던 학생 스케줄 확인 작업을 엑셀 다운로드와 자동화 버튼 클릭으로 줄였습니다.
- 전체 처리 시간이 약 1~2분 수준으로 단축되었습니다.
- 단과와 타학원 일정 충돌을 즉시 확인할 수 있게 되었습니다.
- 학생 부재 사유를 더 빠르게 파악할 수 있어 학부모 응대 품질이 개선되었습니다.
- 학생 관리 관련 불만과 컴플레인이 감소했습니다.

## Recognition

이 프로젝트는 소속 교육기업의 전사 AI 활용 우수사례 TOP 6에 선정되었습니다. 이후 사내 협업 플랫폼의 AI 활용 우수사례 콘텐츠로 인터뷰가 게시되었고, 우수사례 상금을 수상했습니다.

## Privacy

포트폴리오 공개를 위해 실제 학생 이름, 학번, 학원명, 출석 시트 ID 등 민감한 정보는 모두 비식별 처리했습니다. 공개용 Apps Script 코드에서도 외부 출석 시트 ID는 `YOUR_ATTENDANCE_SPREADSHEET_ID` placeholder로 대체했습니다.
