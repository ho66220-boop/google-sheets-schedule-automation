# Sheet Structure

이 문서는 스케줄 자동화 파일의 주요 시트 역할을 정리합니다. 공개용 샘플에서는 학생 이름, 학번, 학원명 등 민감한 정보가 비식별 처리되어 있습니다.

## 기본

학생의 기본 정보를 관리하는 시트입니다.

주요 데이터:

- 학번
- 학년
- 성별
- 호실
- 좌석
- 학반 또는 분류 코드

이 시트의 정보는 교실별 시트에서 학생이 어느 호실과 좌석에 배정되어 있는지 판단하는 기준으로 사용됩니다.

## 단과수강생

관리 시스템에서 다운로드한 엑셀 데이터를 붙여넣는 시트입니다.

주요 데이터:

- 학번
- 이름
- 학년
- 수강기호
- 강사
- 과목
- 수강 시작일
- 종강일
- 휴강일

Apps Script는 이 시트의 수강기호를 파싱하여 요일과 교시를 해석합니다. 시간표 자동화에서는 A:F 범위만 사용하고, 출석 반영에서는 G:I의 날짜 조건까지 함께 사용합니다.

## 타학원생 관리

단과 외 고정 일정을 관리하는 시트입니다.

예시 일정:

- 타학원 수업
- 정기 외출
- 반복 지각
- 기타 고정 부재 사유

직원마다 입력 방식이 달라질 수 있기 때문에, Apps Script에서는 요일 범위와 교시 범위를 유연하게 해석하도록 구성했습니다.

## 교실별 시트

실제 주간 시간표가 생성되는 시트입니다.

예시:

- `301호`
- `401호`
- `2관301호`
- `2관402호`

각 교실별 시트에는 학생의 학번, 좌석, 교시가 있고, Apps Script가 H:N 영역에 월요일부터 일요일까지의 주간 시간표를 생성합니다.

## 충돌확인

단과 수업과 정기 일정이 같은 요일/교시에 겹치는 경우 자동으로 기록되는 시트입니다.

기록 항목:

- 교실
- 좌석
- 요일
- 교시
- 기존 내용
- 충돌 내용

이 시트를 통해 담당자는 학생에게 잘못된 스케줄을 빠르게 안내할 수 있습니다.

## 시간표(본관), 시간표(2관)

학생에게 보기 좋게 출력하기 위한 시간표 시트입니다. 교실별 시트에 생성된 데이터를 바탕으로 학생별 스케줄을 확인하는 용도로 사용합니다.

## 출석 시트

실제 출결이 이루어지는 외부 시트입니다.

공개용 코드에서는 실제 출석 시트 ID를 제거하고 `YOUR_ATTENDANCE_SPREADSHEET_ID`로 대체했습니다. 실제 운영 환경에서는 해당 값을 운영용 출석 시트 ID로 설정해야 합니다.
