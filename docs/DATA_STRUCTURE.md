# Data Structure

이 문서는 주요 시트의 역할과 데이터 흐름을 정리합니다. 공개용 샘플에서는 실제 학생 정보와 기관명, 외부 시트 ID를 비식별 처리했습니다.

## 주요 시트

| 시트 | 역할 |
|---|---|
| 기본 | 학생의 학번, 학년, 호실, 좌석 등 기본 정보 관리 |
| 단과수강생 | 관리 시스템에서 다운로드한 수강생 데이터를 입력하는 시트 |
| 타학원생 관리 | 타학원, 정기 외출 등 단과 외 반복 일정 관리 |
| 교실별 시트 | 학생별 주간 시간표 생성 결과 확인 |
| 충돌확인 | 단과와 정기 일정이 겹칠 때 기록 |
| 시간표(본관), 시간표(2관) | 학생에게 보여주기 위한 출력용 시간표 |
| 출석 시트 | 실제 출결이 이루어지는 외부 시트 |

## 단과수강생 시트

Apps Script는 `단과수강생` 시트에서 수강 정보를 읽습니다.

| 컬럼 | 의미 |
|---|---|
| A | 학번 |
| B | 이름 |
| C | 학년 |
| D | 수강기호 |
| E | 강사 |
| F | 과목 |
| G | 개강일 또는 수강 시작일 |
| H | 종강일 |
| I | 휴강일 |

시간표 생성 단계에서는 A:F만 사용합니다. 날짜 조건은 전체 주간 시간표를 만들 때 적용하지 않습니다.

출석 반영 단계에서는 A:I를 사용합니다. 이 단계에서 개강일, 종강일, 휴강일 조건을 반영합니다.

## 타학원생 관리 시트

타학원, 정기 외출, 반복 지각 등 단과 외 일정을 관리합니다.

주요 입력값:

- 호실
- 좌석
- 요일/교시 코드
- 사유

직원마다 입력 방식이 달라질 수 있으므로, 스크립트는 한글 요일, 요일 범위, 교시 범위를 유연하게 해석하도록 설계했습니다.

## 데이터 흐름

```txt
관리 시스템 엑셀
  ↓
단과수강생 시트
  ↓
수강기호 파싱
  ↓
교실별 시간표 생성
  ↓
충돌확인 시트 기록
  ↓
출석 시트 반영
```

## 보호값

출석 시트에는 운영자가 직접 입력한 값이 있습니다. 자동화가 이런 값을 덮어쓰면 실제 출결 기록이 훼손될 수 있으므로 보호값 목록을 별도로 두었습니다.

예시:

- 지각
- 병결
- 조퇴
- 외출
- 학교
- 타학원
- 결석

자동화는 보호값을 발견하면 해당 셀을 수정하지 않습니다.
