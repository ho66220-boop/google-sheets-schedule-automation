# Workflow

이 문서는 자동화 도입 전후의 업무 흐름을 정리합니다. 목적은 코드 설명보다, 실제 운영자가 어떤 반복 작업을 줄였는지 보여주는 것입니다.

## 기존 업무 흐름

1. 담당자가 학원 관리 시스템에 접속합니다.
2. 학생 이름을 검색해 단과 수강 여부를 확인합니다.
3. 전체 수강생 엑셀을 내려받은 경우, 필요한 학생만 다시 분류합니다.
4. 수강기호를 보고 요일과 교시를 직접 해석합니다.
5. 교실별 시간표에 수기로 입력합니다.
6. 타학원, 정기 외출 등 별도 일정은 담당자별로 따로 관리합니다.
7. 공용 출석 시트에 필요한 내용을 다시 입력합니다.
8. 단과와 타학원 일정이 겹치는지 사람이 확인합니다.

## 개선 후 업무 흐름

1. 관리 시스템에서 수강생 엑셀을 다운로드합니다.
2. `단과수강생` 시트에 데이터를 붙여넣습니다.
3. `타학원생 관리` 시트에 정기 일정과 타학원 일정을 입력합니다.
4. Google Sheets 메뉴에서 시간표 자동화를 실행합니다.
5. 교실별 시간표가 자동 생성됩니다.
6. 일정 충돌은 `충돌확인` 시트에 기록됩니다.
7. 출석 반영 메뉴를 실행합니다.
8. 오늘부터 7일간의 유효 일정이 출석 시트에 반영됩니다.

## 운영상 장점

- 담당자가 학생별로 일일이 검색하는 시간이 줄어듭니다.
- 수강기호 해석 방식이 사람마다 달라지는 문제를 줄입니다.
- 타학원/정기 일정이 공용 구조 안에서 관리됩니다.
- 학생 부재 사유 확인이 빨라집니다.
- 충돌을 사후가 아니라 사전에 확인할 수 있습니다.

## 자동화 범위

자동화가 처리하는 범위:

- 수강기호 파싱
- 교실별 시간표 생성
- 타학원/정기 일정 반영
- 단과와 정기 일정 충돌 기록
- 오늘부터 7일간의 출석 시트 반영
- 수기 입력 보호값 유지

자동화가 대신하지 않는 범위:

- 관리 시스템에서 엑셀을 다운로드하는 작업
- 실제 학생 상담 또는 학부모 응대 판단
- 충돌 발생 후 최종 스케줄 조정
- 운영자가 직접 입력한 출결 사유의 의미 판단
