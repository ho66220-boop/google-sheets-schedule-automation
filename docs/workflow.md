# Workflow

## 1. 관리 시스템 엑셀 다운로드

학원 관리 시스템에서 전체 수강생 엑셀 파일을 다운로드합니다. 기존에는 이 데이터를 사람이 직접 분류하고 수강기호를 해석했지만, 자동화 이후에는 시트에 붙여넣는 방식으로 입력 단계를 단순화했습니다.

## 2. 단과수강생 시트에 데이터 입력

다운로드한 엑셀 데이터를 `단과수강생` 시트에 붙여넣습니다.

Apps Script는 다음 정보를 읽습니다.

- 시간표 생성: A:F
- 출석 반영: A:I

시간표 생성 단계에서는 개강일, 종강일, 휴강일을 보지 않습니다. 전체 주간 시간표 구조를 먼저 만들기 위함입니다.

## 3. 타학원 및 정기 일정 입력

단과 외 일정은 `타학원생 관리` 시트에 입력합니다.

입력 예시:

- 특정 요일 특정 교시의 타학원 수업
- 반복 외출
- 정기 조퇴

직원마다 입력 방식이 조금씩 다를 수 있기 때문에, 스크립트는 한글 요일, 요일 범위, 교시 범위 등을 파싱할 수 있도록 구성했습니다.

## 4. 시간표 자동화 실행

Google Sheets 메뉴의 `시간표 자동화`에서 실행합니다.

실행 결과:

- 교실별 시트 H:N에 월~일 시간표 생성
- 단과 일정 반영
- 타학원 및 정기 일정 반영
- 충돌 발생 시 `충돌확인` 시트에 기록

## 5. 충돌 확인

`충돌확인` 시트에서 단과와 정기 일정이 겹치는 항목을 확인합니다.

이 단계는 학생에게 스케줄 조정이 필요하다는 안내를 빠르게 하기 위한 장치입니다.

## 6. 출석 시트 반영

Google Sheets 메뉴의 `출석 반영`에서 실행합니다.

출석 반영 단계에서는 오늘부터 7일간의 날짜 컬럼만 대상으로 삼습니다. 이때 `단과수강생` 시트의 개강일, 종강일, 휴강일 조건을 적용합니다.

반영 규칙:

- 개강일 이전이면 자동값을 입력하지 않음
- 종강일 이후이면 기존 자동값 삭제
- 휴강일이면 기존 자동값 삭제
- 직원이 직접 입력한 보호값은 덮어쓰지 않음

## 7. 결과 확인

출석 시트에 자동 반영된 내용을 확인합니다.

이 흐름을 통해 기존에 학생별로 검색하고 수기로 옮기던 작업을 엑셀 다운로드와 버튼 클릭 중심의 작업으로 줄였습니다.
