# Test Cases

이 문서는 실제 운영 과정에서 확인해야 하는 주요 테스트 케이스를 정리합니다.

## 1. 수강기호 파싱

| 케이스 | 입력 예시 | 기대 결과 |
|---|---|---|
| 단일 요일/블록 | `A1` | 월요일 1블록에 해당하는 교시 반영 |
| 복수 요일/블록 | `ACE2` | 월/수/금 2블록 반영 |
| 잘못된 코드 | 빈값 또는 형식 불일치 | 시간표에 반영하지 않음 |

## 2. 타학원 및 정기 일정

| 케이스 | 입력 예시 | 기대 결과 |
|---|---|---|
| 특정 요일 특정 교시 | `월3` | 월요일 3교시에 사유 반영 |
| 요일 범위 | `월~수3` | 월/화/수 3교시에 사유 반영 |
| 교시 범위 | `금2~4` | 금요일 2~4교시에 사유 반영 |
| 쉼표 분리 입력 | `화12,수34` | 각 요일/교시에 나누어 반영 |

## 3. 충돌 감지

| 케이스 | 기대 결과 |
|---|---|
| 단과와 타학원 일정이 같은 요일/교시에 존재 | `충돌확인` 시트에 기록 |
| 기존 값과 새 값이 동일 | 충돌로 기록하지 않음 |
| 일정이 서로 다른 요일/교시에 존재 | 충돌 없음 |

## 4. 날짜 예외

| 케이스 | 기대 결과 |
|---|---|
| 개강일 이전 | 출석 시트에 자동값 입력하지 않음 |
| 종강일 이후 | 기존 자동값 삭제 |
| 휴강일 | 기존 자동값 삭제 |
| 날짜 조건 없음 | 기본 주간 시간표 기준으로 반영 |

## 5. 수기 입력 보호값

| 현재 셀 값 | 기대 결과 |
|---|---|
| 지각 | 자동화가 덮어쓰지 않음 |
| 병결 | 자동화가 덮어쓰지 않음 |
| 조퇴 | 자동화가 덮어쓰지 않음 |
| 빈칸 | 유효한 자동 시간표값 입력 |
| 기존 자동값 | 최신 자동 시간표값으로 갱신 |

## 6. 출석 시트 반영 범위

| 케이스 | 기대 결과 |
|---|---|
| 오늘부터 7일 안의 날짜 컬럼 | 반영 대상 |
| 7일 범위 밖 날짜 컬럼 | 반영하지 않음 |
| 날짜 헤더가 해석되지 않음 | 반영하지 않음 |

## 확인 필요

공개용 저장소에서는 실제 출석 시트 ID를 제거했기 때문에, 외부 출석 시트 반영은 운영 환경에서 별도 확인이 필요합니다.
